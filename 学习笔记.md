## 第一阶段：基础理论储备

### 1.1 脑机接口(BCI)概述

#### 什么是脑机接口？
脑机接口(Brain-Computer Interface, BCI)是一种直接连接大脑与外部设备的技术系统。它绕过传统的肌肉输出通道，实现大脑与计算机或外部设备的直接通信。

#### BCI系统组成
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   信号采集   │ ──▶ │  信号处理   │ ──▶ │  特征解码   │ ──▶ │  输出控制   │
│ (EEG/EMG)  │     │ (滤波/去噪) │     │ (ML/DL)    │     │ (设备控制)  │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

#### BCI类型
| 类型 | 信号来源 | 应用场景 |
|------|----------|----------|
| 侵入式 |皮质脑电 | 医疗康复 |
| 半侵入式 | 硬膜外信号 | 高精度控制 |
| 非侵入式 | EEG/MEG | 消费级BCI |

### 1.2 运动想象(Motor Imagery)

#### 生理基础
- **运动皮层**: 位于大脑中央前回，负责运动规划和执行
- **Mu节律**: 8-13Hz，与运动皮层活动密切相关
- **Beta节律**: 13-30Hz，运动准备和执行时变化显著

#### ERD/ERS现象
- **ERD (Event-Related Desynchronization)**: 事件相关去同步
  - 运动想象时，对侧运动皮层激活
  - Mu/Beta节律幅度降低
  
- **ERS (Event-Related Synchronization)**: 事件相关同步
  - 休息期间，同侧半球
  - Mu/Beta节律幅度增加

#### 运动想象范式
| 任务 | 激活区域 | 脑区 |
|------|----------|------|
| 左手想象 | 右侧运动皮层 | C4 |
| 右手想象 | 左侧运动皮层 | C3 |
| 脚想象 | 顶部运动皮层 | Cz |

## 第二阶段：信号处理与特征工程

### 2.1 EEG信号特点

#### 优点
- 无创采集
- 时间分辨率高(毫秒级)
- 设备相对便宜
- 便携性好

#### 挑战
- 空间分辨率低
- 信号微弱(μV级)
- 易受伪影干扰
- 个体差异大

### 2.2 预处理流程

#### 2.2.1 陷波滤波(Notch Filter)
```
目的: 去除50Hz工频干扰
原理: 在50Hz处创建窄带阻滤波器
参数: 品质因子Q=30-35
```

#### 2.2.2 带通滤波(Bandpass Filter)
```
目的: 提取有效频段
常用频段:
- Mu节律: 8-13Hz
- Beta节律: 13-30Hz
- 全频段: 0.5-40Hz
```

#### 2.2.3 伪影处理
| 伪影类型 | 来源 | 处理方法 |
|----------|------|----------|
| 眼动 | 眨眼/眼动 | ICA/ADJUST |
| 肌电 | 肌肉收缩 | 高通滤波 |
| 心电 | 心脏跳动 | ICA/模板相减 |

### 2.3 特征提取方法

#### 时域特征
- 均值、方差
- 峰值、峰峰值
- 功率谱密度

#### 频域特征
- 频带功率
- 谱熵
- 频谱重心

#### 空域滤波
- **CSP (Common Spatial Pattern)**: 寻找最大化类间方差的投影方向
- **LDA (Linear Discriminant Analysis)**: 线性判别分析

## 第三阶段：机器学习算法

### 3.1 PCA主成分分析

#### 核心思想
PCA通过线性变换，将高维数据投影到低维空间，同时最大化保留原始数据的方差信息。

#### 数学原理
```
1. 中心化: X' = X - μ

2. 协方差矩阵: 
   Cov = (1/(n-1)) X'^T X'

3. 特征分解:
   Cov·v = λ·v
   - λ: 特征值(方差)
   - v: 特征向量(方向)

4. 降维投影:
   Y = X'·V_k
```

#### 应用场景
- 数据降维
- 特征提取
- 可视化
- 噪声去除

### 3.2 Hopfield神经网络

#### 网络结构
```
特点:
- 全互联: 每个神经元连接其他所有神经元
- 二值状态: +1 或 -1
- 权值对称: W_ij = W_ji
- 无自连接: W_ii = 0
```

#### Hebb学习规则
```
原理: "一起激活的神经元，连接加强"

公式: W_ij = (1/n) Σ x_i · x_j

其中:
- x_i, x_j: 神经元状态
- n: 训练样本数
```

#### 联想记忆机制
```
工作流程:
1. 存储: 将模式通过Hebb规则存入权值矩阵
2. 检索: 输入部分/噪声模式
3. 收敛: 异步更新直至稳定
4. 输出: 收敛到最近存储的模式
```

#### 能量函数
```
E = -½ Σ Σ w_ij s_i s_j

性质:
- 能量随状态更新单调递减
- 收敛点对应局部最小
- 存储模式对应能量局部最小
```

### 3.3 分类器对比

| 分类器 | 优点 | 缺点 | 适用场景 |
|--------|------|------|----------|
| Hopfield | 无监督、简单 | 存储容量有限 | 小样本模式识别 |
| SVM | 泛化能力强 | 大数据慢 | 分类边界清晰 |
| CNN | 特征自学习 | 需要大量数据 | 复杂模式 |
| LSTM | 时序建模 | 训练复杂 | 时序EEG |

### 4.2 异步编程

#### async/await
```python
# 同步vs异步
def fetch_data():  # 同步，阻塞
    response = requests.get(url)
    return response.json()

async def fetch_data():  # 异步，非阻塞
    async with httpx.AsyncClient() as client:
        response = await client.get(url)
        return response.json()
```

#### 适用场景
- I/O密集型: API调用、文件读写
- 高并发: 大量并发请求
- 实时通信: WebSocket

### 5.1 Vue3框架

#### Composition API
```typescript
// 选项式 API
export default {
  data() {
    return { count: 0 }
  },
  methods: {
    increment() {
      this.count++
    }
  }
}

// 组合式 API
import { ref } from 'vue'
export default {
  setup() {
    const count = ref(0)
    const increment = () => count.value++
    return { count, increment }
  }
}
```

```
浏览器 ──▶ Nginx ──▶ FastAPI ──▶ Python算法
  ◀── JSON ──▶        ◀── 响应 ───┘
```

#### 优势
- 独立开发/部署
- 技术栈灵活
- 更好的扩展性

### 6.2 部署方案

#### 开发环境
```bash
# 后端
uvicorn app.main:app --reload

# 前端
npm run dev
```

#### 生产环境
```bash
# 前端构建
npm run build

# Nginx配置
server {
    location / {
        root /var/www/dist;
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://localhost:8000;
    }
}
```

### 6.3 Docker部署

```dockerfile
# Dockerfile
FROM python:3.10
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0"]
```
